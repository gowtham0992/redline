# Internet Dogfood Sources

This is the public-data shortlist for redline dogfooding. The goal is not to
train a model. The goal is to get realistic prompt-response logs, generate a
suite, replay or compare candidates, and verify that redline catches concrete
behavioral regressions.

Do not commit downloaded dataset rows unless the license and privacy posture
explicitly allow redistribution. Prefer local samples under `.redline/private/`
or generated fixtures under `examples/` that are synthetic and source-inspired.

## What redline needs

redline needs two things:

- A baseline JSONL with `prompt` and `response`.
- A candidate JSONL for the same prompts, or a replay runner that can regenerate
  candidate responses from a changed prompt/model.

Most internet datasets provide the baseline side. Use them in one of three ways:

- Baseline from dataset, candidate from your changed model or prompt runner.
- Baseline from one model/provider column, candidate from another model/provider
  column when the dataset has multi-model responses.
- Baseline from dataset, candidate from a deterministic degradation script for
  release-safety checks. This is useful for product QA, but label it synthetic.

## Best Sources

| Source | Best use | License posture | Redline fit |
| --- | --- | --- | --- |
| Databricks Dolly 15k | First internet dogfood run | CC BY-SA 3.0 | Direct `instruction/context/response` rows; small enough to sample locally. |
| OpenAssistant OASST1 | Multilingual assistant-tree dogfood | Apache 2.0 | Real prompter/assistant message trees, ratings, and many languages. |
| WildChat | Real-world chat-shape dogfood | ODC-By / AI2 terms; handle privacy carefully | Real user-chatbot conversations; good for broad prompt variety and long-tail formats. |
| LMSYS-Chat-1M | High-realism gated dogfood | Dataset agreement; do not redistribute | OpenAI-style conversations, model names, moderation tags; useful for local private dogfood. |
| Stanford Alpaca | Research-only baseline prompt variety | CC BY-NC 4.0 | Instruction/input/output shape, but non-commercial. Do not use for public product fixtures. |

## Recommended Order

1. **Databricks Dolly 15k**: easiest direct JSONL conversion.
2. **OpenAssistant OASST1**: tests tree normalization and multilingual behavior.
3. **WildChat**: tests real-world chat messiness after privacy review.
4. **LMSYS-Chat-1M**: use only after accepting dataset terms; keep samples local.
5. **Alpaca**: use only for private research dogfood.

## Dolly 15k Recipe

Dolly rows already look like prompt-response logs:

- `instruction`
- `context`
- `response`
- `category`

Suggested local conversion:

```bash
python -m pip install datasets
python - <<'PY'
import json
from datasets import load_dataset

rows = load_dataset("databricks/databricks-dolly-15k", split="train")
with open(".redline/private/dolly-baseline.jsonl", "w", encoding="utf-8") as out:
    for item in rows.select(range(250)):
        context = (item.get("context") or "").strip()
        instruction = (item.get("instruction") or "").strip()
        prompt = instruction if not context else f"{instruction}\n\nContext:\n{context}"
        out.write(json.dumps({
            "prompt": prompt,
            "response": item.get("response") or "",
            "metadata": {"source": "databricks-dolly-15k", "category": item.get("category")},
        }, ensure_ascii=False) + "\n")
PY

redline suite .redline/private/dolly-baseline.jsonl \
  --out .redline/private/dolly-suite.json \
  --max-cases 80
```

Then compare a candidate generated by your runner:

```bash
redline eval .redline/private/dolly-suite.json \
  --replay "python runners/stdio_runner.py" \
  --fail-on none
```

What to watch for:

- Lost list/table/code structure in generation tasks.
- Missing extracted entities or numbers in information extraction tasks.
- Content drift in summarization tasks.

## OpenAssistant OASST1 Recipe

OASST1 is a message-tree dataset. Build prompt-response pairs by joining a
prompter node with one assistant child.

Suggested approach:

- Filter `deleted=false`.
- Prefer rows where `role=prompter` and the assistant child has positive
  quality labels or rank.
- Preserve `lang` in metadata so redline multilingual refusal checks can be
  exercised.

Useful dogfood goals:

- Compare English vs non-English behavior.
- Verify redline does not overfit English-only refusal phrasing.
- Exercise long assistant answers where `--profile review` is safer than strict
  blocking.

## WildChat Recipe

WildChat is useful because it looks like real app traffic: messy prompts,
multiple languages, long conversations, moderation metadata, and broad task
variety.

Use it as private/local dogfood:

- Sample non-toxic rows first.
- Keep raw rows under `.redline/private/`.
- Convert only one user turn and the following assistant turn into each redline
  prompt-response row.
- Redact or discard rows containing private data before sharing screenshots or
  case studies.

Good redline checks:

- Empty output.
- Refusal shifts.
- Lost Markdown/code/list/table format.
- Large content drift in same-shape long-form answers.

## LMSYS-Chat-1M Recipe

LMSYS-Chat-1M is high-value but gated and governed by a dataset agreement.
Use it only after accepting the terms, and do not redistribute samples.

Why it matters:

- Conversations are already in OpenAI API-style JSON.
- Rows include model name, detected language, and moderation tags.
- It can support baseline-vs-candidate comparisons across model families.

Safe redline use:

- Keep samples private.
- Sample by language and moderation tag.
- Use model A as baseline and model B as candidate only when prompts align.
- Do not commit raw rows, generated suites, or reports containing original text.

## Stanford Alpaca Recipe

Alpaca is useful for local research-only prompt variety, but its data is
non-commercial. Do not use it for public product fixtures or launch claims.

If used privately:

- Treat `instruction + input` as prompt.
- Treat `output` as baseline response.
- Label any derived fixture as research-only.

## What We Should Build Next

These sources point to two useful future redline adapters:

- `jsonl_log_adapter.py --preset dolly`
- `jsonl_log_adapter.py --preset oasst1`

WildChat and LMSYS should stay documented first because their privacy and terms
need more care before we make copy-paste import commands.

## Public Case Study Policy

Before turning internet dogfood into a public case study:

- Confirm license and attribution requirements.
- Link the original dataset page in the write-up.
- Do not publish raw third-party rows unless redistribution is clearly allowed.
- Prefer synthetic fixtures inspired by public dataset patterns.
- Include source links and say exactly what is synthetic.
- Keep private samples in `.redline/private/`.
